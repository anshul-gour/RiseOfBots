Each team can have only 2 participants.

What are the soft wares needed to run the game?

For Windows, Microsoft Visual  Studio6 along with OpenGL libraries needed. In Linux, any C/C++ compiler will work but only if you have Mesa v5 or later version installed.

Can a team submit two driving programs?

A team can  submit only one program. A person can be a member of only one team.

2. What is gametime?

The player's program is executed every gametime. The combat server checks for requests from each of the bots every game time and processes it accordingly (This processing won't take much time. It just involves changing some variables). The player cannot do anything between two gametimes.

3. Does the code work in Windows as well as Linux?

Programming can be done in Windows as well as Linux.  The participant's code is OS independent. Only the Graphics library used will be different( for .more related to OS, refer to installation page

4. Does Player0 get any advantage like first move ?

No. The combat server takes inputs from driving programs  of all the bots. After this process is over, it moves (processes requests from) all the bots simultaneously.

5. When is a edan deactivated?

When an edan makes a request which the combat server cannot understand, the edan disqualifies. Its team is penalized and any further requests from the bot's driving program are ignored.
